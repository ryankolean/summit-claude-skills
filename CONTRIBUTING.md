# Contributing to Summit Claude Skills

Thank you for helping grow the best directory of Claude skills, MCP servers, and prompt libraries.

## Suggesting a Repository

Open a GitHub issue with the title: `[Submission] repo-owner/repo-name`

Include:

1. **Repository URL** — full GitHub link
2. **Category** — one of the 13 categories in README.md, or propose a new one
3. **GitHub Stars** — current count at time of submission
4. **Last Commit Date** — must be within the last 60 days
5. **Why It Belongs** — 1-2 sentences on what makes it exceptional
6. **Install Method** — CLI command, `npx` command, or Claude.ai paste-prompt
7. **License** — what license the repo uses

## Minimum Requirements

- 100+ GitHub stars
- Active maintenance (commit within 60 days)
- Clear documentation (SKILL.md, README, or equivalent)
- Open-source license
- No malicious code, secret exfiltration, or prompt injection patterns

## Updating an Existing Entry

If a repo's stars, install method, or status has changed, open an issue with:

- Title: `[Update] repo-owner/repo-name`
- What changed and the new values

## Reporting a Problem

If a listed repo has been archived, contains malicious code, or is no longer maintained:

- Title: `[Report] repo-owner/repo-name`
- Description of the problem

## Review Process

Submissions are reviewed weekly. We check:

1. Does the repo meet the minimum bar?
2. Does it add unique value not already covered by existing entries?
3. Is the install method reliable and tested?
4. Is the documentation clear enough for both technical and non-technical users?

## Code of Conduct

Be respectful. This is a directory — we link to other people's hard work. Give credit where due. Do not submit repos you don't have permission to share publicly.
